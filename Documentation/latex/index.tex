A simple command line tool for encrypting/decrypting text using classical ciphers\hypertarget{index_autotoc_md2}{}\doxysection{Building $<$tt$>$mpags-\/cipher$<$/tt$>$}\label{index_autotoc_md2}
Compilation of {\ttfamily mpags-\/cipher} requires the \href{http://www.cmake.org}{\texttt{ CMake}} build tool, plus a C++11 compatible compiler (GCC 4.\+8 or better, Clang 3.\+4 or better are recommended) and {\ttfamily make} on a UNIX operating system. Windows platforms with Visual Studio 2015 or better are also expected to work, but not tested.

To build from a clone of this repository, open a terminal window and change directory into that holding this README. Create a build directory in which to run {\ttfamily cmake} and the build, and change into it\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\$ ls}
\DoxyCodeLine{CMakeLists.txt   LICENSE          MPAGSCipher      README.md        mpags-\/cipher.cpp}
\DoxyCodeLine{\$ mkdir ../Build}
\DoxyCodeLine{\$ cd ../Build}

\end{DoxyCode}


Run {\ttfamily cmake} in this directory, pointing it to the directory holding this README, and consequently the top level CMake script for the project\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\$ cmake ../<this dir>}
\DoxyCodeLine{... system specific output ...}
\DoxyCodeLine{-\/-\/ Configuring done}
\DoxyCodeLine{-\/-\/ Generating done}
\DoxyCodeLine{-\/-\/ Build files have been written to: ... your build dir path ...}
\DoxyCodeLine{\$}

\end{DoxyCode}


The exact output will depend on your system, compiler and build directory location, but you should not see any errors. CMake will generate Makefiles by default on UNIX platforms, so to build, simply run {\ttfamily make} in the build directory\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\$ ls}
\DoxyCodeLine{CMakeCache.txt      CMakeFiles          Makefile            cmake\_install.cmake}
\DoxyCodeLine{\$ make}
\DoxyCodeLine{... verbose output ...}
\DoxyCodeLine{[100\%] Built target mpags-\/cipher}
\DoxyCodeLine{...}
\DoxyCodeLine{\$}

\end{DoxyCode}

If no input file is supplied, {\ttfamily mpags-\/cipher} will wait for user input from the keyboard until RETURN followed by CTRL-\/D are pressed. It will then echo the input to stdout or write it to the file supplied with the {\ttfamily -\/o} option.

To ensure the input text can be used with the character sets known to classical ciphers, it is transliterated using the following rules\+:


\begin{DoxyItemize}
\item Alphabetical characters are converted to uppercase
\item Digits are translated to their English equivalent words (e.\+g. \textquotesingle{}0\textquotesingle{} -\/$>$ \char`\"{}\+ZERO\char`\"{})
\item All other characters (punctuation) are discarded
\end{DoxyItemize}

At present, the Caesar, Playfair and Vigenere ciphers are supported.\hypertarget{index_autotoc_md3}{}\doxysection{Testing}\label{index_autotoc_md3}
